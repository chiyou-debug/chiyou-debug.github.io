    # Oracle

    # 1. oracle介绍

    # 1.1 数据库

    • 使用io流文件存储数据的弊端

      1. 文件存储数据存储效率低；
      2. 不管存还取操作比较麻烦;
      3. 一般只能保存小量字符串数据等;
      4. 不能进行跨主机访问

    • 数据库

      DB的全称是Data Base，即数据库的意思。数据库实际上就是一个文件集合，是一个存储数据的仓库，本质就是一个文件系统，数据库是按照特定的格式把数据存储起来，用户可以对存储的数据进行增删改查操作；

    • 数据库管理系统

    DBMS的全称是Database Management System，即数据库管理系统的意思，是一个软件，用来管理数据库文件的软件，用户可以访问DBMS对数据进行增删改查操作，常见DBMS有： MySQL、Oracle、DB2、SQLite、SQLServer等

    • 主流关系型数据库简介
    1. MySQL： 08年被sun公司收购，09年sun被oracle收购，开源免费，到oracle发布了5.0版本(使用了oracle核心技术 性能提高30%)，因为Oracle数据库地位受到威胁，计划把mysql闭源，原mysql作者们不干了，原程序员出去单干，发布了MariaDB数据库、名字是因为作者女儿叫Maria ，市场排名第一；
    2. Oracle：闭源、最贵、性能最高，市场排名第二；
    3. SQLServer：微软公司的产品，市场排名第三，主要应用在.net(C#)开发的网站中；
    4. DB2：IBM公司有做网站的完整解决方案（操作系统，we服务器（如tomcat），数据库等）主要应用在银行等国有大型企业中；
    5. SQLite：轻量级数据库，只有几十k，一般应用在嵌入式和移动设备中；
    • 开源和闭源的区别

    • 开源：开发源代码，盈利方式靠卖服务，赚钱慢少，用户群大，大牛程序员会无偿帮着升级维护；

    • 闭源：不开放源代码，盈利方式卖产品+服务，赚钱快多，大牛们各种攻击找漏洞；

    # 1.2 oracle组成

    • 物理结构

      Oracle服务器主要由实例、数据库、程序全局区和前台进程组成，其中:

      • 实例就是用来提供管理数据库的功能；
        • 进程+内存（操作系统的）

      实例可以进一步划分为系统全局区(SGA)和后台进程(PMON、SMON等)两部分，其中:

      • SGA是所有用户进程共享的一块内存区域，使用操作系统的内存资源;

      • 后台进程需要使用CPU与内存资源。

      • 数据库由数据文件、控制文件和重做控制文件组成，用来存储系统数据，数据存放在硬盘中。

      • 程序全局区(PGA)是一个非共享的内存区域，用于管理用户进程的私有资源。

      • 前台进程可以再划分为用户进程和服务器进程，他们需要使用CPU与内存资源。

    • 逻辑结构

      逻辑结构由表空间(tablespace)、段(segment)、区间(extent)、数据块(data block)组成 。

      • 表空间：存储数据库表

        表空间是一个逻辑容器，它位于逻辑存储结构的顶层。表空间内的数据被物理存放在数据文件中。一个表空间可以包含一个或多个数据文件。但是一个数据文件只能属于一个表空间。

      • 段：

        段是占用存储空间的数据库对象，用于存储和隔离不同数据库对象的数据。

      • 区间：

        区间是Oracle存储空间最小的分配单位。

      • 数据块：

        数据块是Oracle最小的I/O单位。

    # 2.4 锁定或解锁用户

    通常我们遇到oracle用户密码输入多次错误或者长时间不登录之后，那么用户就会变成锁定状态 。

    -- 对用户加锁/解锁 操作用户必须要有对应的权限才可以做这些操作
-- 解锁语法: alter user 用户名 account unlock;
-- 加锁语法: alter user 用户名 account lock;

SQL> show user;
USER is "SCOTT"
SQL> alter user scott account lock;
alter user scott account lock
*
ERROR at line 1:
ORA-01031: insufficient privileges	-- scott用户权限不够


SQL> conn / as sysdba
Connected.
SQL> alter user scott account lock;

User altered.

SQL> alter user scott account unlock;          

User altered.

    # 3. SQL

    sql是一门语言

    通过sqlplus工具, 在这个工具中执行sql语句就可以操作数据库数据了

    sql语句中的关键字不区分大小写

    # 3.1 sql语言类型

    • DML -- 数据操纵语言 （Data Manipulation Language）
      • insert （添加数据）
      • delete （删除数据）
      • update（修改数据）
      • select （查询数据）
    • DDL -- 数据定义语言（Data Definition Language）
      • create table （创建表）
      • alter table （修改表）
      • truncate table（清空表）
      • drop table （删除表）
      • create view （视图）
      • create index （索引）
      • create sequence（序列）
      • create synonym（同义词）
    • DCL -- 数据控制语言（Data Control Language）
      • commit （提交）
      • rollback （回滚）

    -- 语法描述说明：
	-- 花括号{}括起的部分为必填部分
	-- 中括号[]括起来的部分为可选部分
	-- 多种形式的语法用竖线|来表示并列单选，或者的意思

		[ORDER BY column1[,column2,…]];

    # 4.1 查询所有列

    SELECT	* FROM 表名;
-- * 表示所有的列

    查询 "emp" 表中所有员工的所有的记录。

    select * from emp;

    # 4.2 查询指定列

    SELECT	列名1|表达式, 列名2|表达式(1+2), ... FROM 表名;

    查询"emp" 表中员工号, 姓名, 月薪, 佣金, 年薪。

    SQL> select empno, ename, sal, comm from emp;
SQL> select empno, ename, sal, comm, sal*12 from emp;

    # 4.3 给列指定别名

    -- as是可以省略的
-- 别名如果是字符串中间如果有空格, 别名需要加""
SELECT	列名1|表达式 as 别名, 列名2|表达式(1+2) 别名, ... FROM 表名;

    要求: 给上述查询出的列名(字段名)指定别名

    查询"emp" 表中员工号, 姓名, 月薪,  佣金, 年薪。
SQL> select empno, ename, sal 工 资, comm as 佣金, sal*12 年薪 from emp;

    # 4.4 表达式中的空字段

    查询员工号, 姓名, 月薪, 佣金, 年薪, 总收入。 [总收入 = 年薪 + 佣金]

    -- 如果comm为空, 总收入就为空
-- 如果字段值为空, 不是0, 这个字段参与表达式的计算,这个表达式的值为空NULL
SQL> select empno, ename, sal "工 资", comm as 佣金, sal*12 年薪, sal*12+comm "总收入" from emp;

-- 解决上边的问题, 需要使用一个函数 nvl
-- NVL(expr1, expr2): 
	-- 如果oracle第一个参数为空那么显示第二个参数的值，
	-- 如果第一个参数的值不为空，则显示第一个参数本来的值
SQL> select empno, ename, sal "工 资", comm as 佣金, sal*12 年薪, sal*12+nvl(comm, 0) "总收入" from emp;

    # 4.5 去重查询

    -- DISTINCT去重
SELECT	*|{[DISTINCT] column |expression [alias],...} FROM 表名;

    查询员工表不同部门的编号

    SQL> select distinct deptno from emp;

    查看不同部门的不同工种

    -- 如果distinct后有多个字段, 对这些字段都起作用
SQL> select distinct deptno, job from emp;

    # 5. 带条件的查询 - where

    SELECT	*|{[DISTINCT] column |expression [alias],...}
	FROM 表名 [WHERE expression]

    # 5.1 使用比较运算符

    可以使用的比较运算符: >, <, =, >=, <=, !=, <>, between and

    1. 查询部门ID为20的员工信息

      select * from emp where deptno=20;

    2. 查询员工名字为 Nico Robin 的员工信息

      -- 如果字段的类型是字符串, 给字段指定值需要加单引号
-- 字段的名字不区分大小写, 但是字段的值区分大小写
select * from emp where ename='Nico Robin';

    3. 查询薪水不等于5500员工的信息

      select * from emp where sal != 5500;
select * from emp where sal <> 5500;

    4. 查询入职日期为1981年11月17日的员工信息

      -- 数据库中的日期和字符串类型是可以相互进行隐式转换的
SQL> select * from emp where hiredate='1987-11-17';
select * from emp where hiredate='1987-11-17'

ORA-01861: 文字与格式字符串不匹配
-- 错误原因是日期的个数不对

-- 查看日期格式
sql> select * from v$nls_parameters;
PARAMETER                                                        VALUE
----------------------------- -------------------------------------------------------------
NLS_LANGUAGE                                                     SIMPLIFIED CHINESE
NLS_TERRITORY                                                    CHINA
NLS_CURRENCY                                                     ￥
NLS_DATE_FORMAT                                                  DD-MON-RR -->  默认日期的格式

SQL> select * from emp where hiredate='17-11月-81';

    5. 查询工资介于 1000 - 2000 之间的员工的信息.

      -- >=100 and <=200
SQL> select * from emp where sal>=1000  and  sal<=2000;
SQL> select * from emp where sal between 1000  and  2000;

    # 5.2 使用逻辑运算符

    逻辑运算符包括:

    • 或: or
    • 与: and
    • 非: not

    1. 查询部门ID为10, 或者部门ID为20 的员工信息

      sql> select * from emp where deptno=10 or deptno=20;

    2. 查询部门ID为 20, 并且工资为 5500 的员工信息

      sql> select * from emp where deptno=20 and sal=5500;

    3. 查询1981年2月（含2月）到82年2月（不含2月）入职的员工信息

      -- 1981-2-1  到  1982-1-31

    # 7. 单行函数

    任何东西，只要它能接收输入，对输入进行加工并产生输出，它就可以被称为函数。

    单行函数只对表中的一行数据进行操作，并且对每一行数据只产生一个输出结果。单行函数可以接受一个或多个参数，其产生的输出结果的数据类型可能与参数的数据类型不同。

    单行函数分为五种类型：字符函数、数值函数、日期函数、转换函数、通用函数
