    # 3. FastDFS

    # 3.1 fastDFS介绍

    1. fastDFS概述

      • 是用c语言编写的一款开源的分布式文件系统。
        • 余庆 - 淘宝的架构师
      • 为互联网量身定制，充分考虑了冗余备份、负载均衡、线性扩容等机制，注重高可用，高性能指标
        • 冗余备份: 纵向扩容
        • 线性扩容: 横向扩容
      • 可以很容易搭建一套高性能的文件服务器集群提供文件上传、下载等服务。
        • 能够存储文件
          • 网盘
          • 图床

    2. fastDFS框架中的三个角色

      • 管理者- 追踪器 -> tracker
      • 存储节点(存储文件) -> storage
      • 客户端 -> client
        • 上传文件
        • 下载文件

    3. fastDFS三个角色之间的关系

      总结:

      • 三个角色启动的顺序
        • 先启动追踪器 -> 守护进程
        • 再启动存储节点 -> 守护进程
        • 最后启动客户端 -> 普通进程
      • 关系
        • 追踪器和存储节点
          • 存储节点主动连接追踪器
            • 单独创建一线程干这个事儿
              • 上传状态信息
                • 存储节点的使用容量和剩余容量
                • 文件的同步情况
                • 文件的上传和下载次数
        • 客户端和追踪器
          • 上传文件
            • 客户端主动连接追踪器 -> 询问
              • 哪个存储节点可以存储要上传的文件
              • 得到了对应的存储节点 的 IP和Port
            • 客户端主动连接存储节点
              • 上传文件
              • 存储节点接收文件
              • 接收完毕, 存储节点会生成一个文件ID -> fileID
              • 存储节点将这个fileID发送给客户端
              • 客户端存储这个fileID
                • 下载文件的时候需要使用
          • 下载文件
            • 客户端主动连接追踪器 -> 询问
              • 要下载的文件在哪个存储节点上
              • 得到了对应的存储节点 的 IP和Port
            • 客户端主动连接存储节点
              • 向存储节点发送下载请求
              • 存储节点给客户端发送数据
              • 客户端接收数据并写磁盘

    4. fastDFS集群 - （了解即可）

      1. 追踪器集群
        • 为什么集群？
          • 为了避免单点故障
        • 多个Tracker如何工作？
          • 工作方式: 轮询
        • 如何实现集群？
          • 通过配置文件
            • 客户端的配置文件
            • 存储节点的配置文件
      2. 存储节点集群
        • fastDFS管理存储节点的方式？
          • 通过组的方式进行管理
            • group1, group2, ...
          • 默认只有一个组: group1
        • 集群方式（扩容方式）
          • 横向扩容 -> 容量增加
            • 需要添加一个新的组
              • 默认只有一个组: group1
              • 添加新的组: group2
                • 在该组中添加一台主机
          • 纵向扩容 -> 数据备份, 容量不变
            • 在同一组中添加n台主机
            • 当前组的容量已最小的那个为准
        • 如何实现?
          • 修改配置文件
            • 修改存储节点的配置文件

    # 3.4 fastDFS的启动

    1. 第一个启动追踪器 - 守护进程

      #指定配置文件路径的时候一定要使用绝对路径
#启动
$ fdfs_trackerd /etc/fdfs/tracker.conf
#关闭/停止
$ fdfs_trackerd /etc/fdfs/tracker.conf stop
#重启
$ fdfs_trackerd /etc/fdfs/tracker.conf restart

    2. 第二个启动存储节点 - 守护进程

      #指定配置文件路径的时候一定要使用绝对路径
#启动
$ fdfs_storaged /etc/fdfs/storage.conf
#停止
$ fdfs_storaged /etc/fdfs/storage.conf stop
#重启
$ fdfs_storaged /etc/fdfs/storage.conf restart

    3. 最后启动客户端 - 普通进程

      #这个需要程序猿写,但是余庆有上传下载的例子
#上传
$ fdfs_upload_file /etc/fdfs/client.conf Go.txt 
group1/M00/00/00/rBA7H13zvpWASD3EAAAAklnXj2k861.txt  #->上传成功之后得到的文件id
#下载
$ fdfs_download_file /etc/fdfs/client.conf     group1/M00/00/00/rBA7H13zvpWASD3EAAAAklnXj2k861.txt  #->下载的时候,需要根据这个id去下载

    # 3.5 对file_id的解释

    • group1

      • 文件上传到了存储节点的哪一个组
      • 如果有多个组这个组名可变的

    • M00 - 虚拟目录

      • 和存储节点的配置项有映射, 从storage.conf
        • store_path0=/home/chiyou/yunDisk/fastDFS_log/storage/data
          • M00 = store_path0/data
        • store_path1=/home/chiyou/yunDisk/fastDFS_log/storage/data
          • M01 = store_path0/data

    • 00/00 -> M00对应的实际路径的子目录

      • 实际的路径
      • 可变的

    • wKhS_VlrEfOAdIZyAAAJTOwCGr43848.md

      • 文件名包含的信息
      • 采用Base64编码

        • 包含的字段包括

          • 源storage server Ip 地址
          • 文件创建时间

    • 文件大小
      - 文件CRC32效验码

        - 循环冗余校验

    • 随机数

int upload_file(const char* conffile,const char* uploadFile,char* fileID)
{
	char group_name[FDFS_GROUP_NAME_MAX_LEN + 1];
	ConnectionInfo *pTrackerServer;
	int result;
	int store_path_index;
	ConnectionInfo storageServer;

	if ((result=fdfs_client_init(conffile)) != 0)
	{
		return result;
	}

	pTrackerServer = tracker_get_connection();
	if (pTrackerServer == NULL)
	{
		fdfs_client_destroy();
		return errno != 0 ? errno : ECONNREFUSED;
	}

	*group_name = '\0';
	if ((result=tracker_query_storage_store(pTrackerServer, \
	                &storageServer, group_name, &store_path_index)) != 0)
	{
		fdfs_client_destroy();
		fprintf(stderr, "tracker_query_storage fail, " \
			"error no: %d, error info: %s\n", \
			result, STRERROR(result));
		return result;
	}

	result = storage_upload_by_filename1(pTrackerServer, \
			&storageServer, store_path_index, \
			uploadFile, NULL, \
			NULL, 0, group_name, fileID);
	if (result == 0)
	{
		printf("%s\n", fileID);
	}
	else
	{
		fprintf(stderr, "upload file fail, " \
			"error no: %d, error info: %s\n", \
			result, STRERROR(result));
	}

	tracker_disconnect_server_ex(pTrackerServer, true);
	fdfs_client_destroy();

	return result;
}

int upload_file1(const char* confFile,const char* uploadFile,char* fileID,int size)
{
    int fd[2];
    // 1. 创建匿名管道
    int ret = pipe(fd);	
    if(ret == -1)
    {
        perror("pipe");
        return -1;
    }
    // 2. 创建子进程
    pid_t pid = fork();
    if(pid == 0)
    {
        // 子进程
        // 写管道 关闭读端
        close(fd[0]);
        // 标准输出重定向到管道写端
        dup2(fd[1],STDOUT_FILENO);
        // 调用其他进程
        execlp("fdfs_upload_file","fdfs_upload_file",confFile,uploadFile,NULL);
        perror("execlp");
    }
    else 
    {
        // 父进程,读管道
        close(fd[1]);
        read(fd[0],fileID,size);
        close(fd[0]);

        // 资源回收
        wait(NULL);
    }

    return 0;
}
