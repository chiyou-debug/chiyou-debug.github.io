    # Redis 基础

    # 1. 数据库类型

    # 1.1 基本概念

    1. 关系型数据库 - sql
      • 操作的时候必须要使用sql语句
      • 数据需要存储在数据库表中, 需要建表
      • 数据存储在硬盘中
      • 常见的关系型数据库:
        • mysql
        • oracle
        • sqlserver
        • sqlite -> 文件数据库
    2. 非关系数据库 - nosql
      • redis/memcached
      • 操作不需要sql语句, 使用的一系列的命令
      • 通过键值对的方式存储的
      • 数据默认存储在内存中
      • 可以进行数据的持久化
        • 数据从内存同步到硬盘
      • 效率高

    # 1.2 关系/非关系型数据库搭配使用

    RDBMS: Relational Database Management System

    第一次客户端访问的时候

    • 所有数据存储在 关系型数据库中
    • 服务器需要访问关系型数据库，读取数据
    • 服务器读到数据之后-> 判断(程序猿)
      • 如果数据访问频率很高 -> 写到redis

    第二次或者以后

    • 客户端访问一些数据
    • 如果数据访问频率高
      • 从redis中读

    # 2. Redis

    Redis 是一个开源（BSD许可）的，内存中的数据结构存储系统，它可以用作数据库、缓存和消息中间件。

    # 2.1 基本知识点

    1. 安装包下载

      • 英文官方： https://redis.io/
      • 中文官方： http://redis.cn/

    2. Redis安装

      • make
      • sudo make install

    3. redis中的两个角色

      # 服务器角色: redis数据库管理,提供存储服务
# 客户端角色: 连接访问

# 服务器 - 启动
$ redis-server 		 # 按照默认的配置启动
$ redis-server 配置文件 # 按照用户的配置启动

# 客户端
$ redis-cli # 前提:服务器启动的时候使用的是默认配置
$ redis-cli -p 端口
$ redis-cli -h ip -p 端口 # 远程连接
$ shutdown # 通过客户端关闭服务器

# 客户端的测试命令
$ ping [字符串]
eg: 客户端: ping
    服务器: pong
    客户端: ping hello
    服务器: hello

    4. 支持多种类型的数据结构

      • key - value 组织格式
        • key -> 必须是字符串
      • value
        • string -> 字符串
        • Lists -> 按插入顺序排序的字符串元素的集合。他们基本上就是链表(linked lists)
        • Sets -> 不重复且无序的字符串元素的集合。
        • sorted sets -> 排序集合且不允许重复, 但是每个字符串元素都关联到一个叫score浮动数值。里面的元素总是通过score进行着排序
          • ([score member], [score member], [score member])
        • Hashes -> 由field和关联的value组成的map。field和value都是字符串的。
          • field- value

    补充：hash和map区别

    • 都是键值对，都是通过key值找value值
    • map查找速度很快，因为map底层数据结构是变种红黑树
    • hash底层数据结构是数组

    # 2.2 redis常用命令

    • String类型

      key -> string
value -> string

# 组织键值对
$ SET key value # 组织一个键值对
$ GET key 		# 根据key取value值

$ MSET key value [key value ...] # 设置多个键值对
$ MGET key [key ...]			 # 获取多个key的value值

# 追加值
$ APPEND key value # 在value值的后边追加新的字符串

# 增减值
$ DECR key 	# 将 key 中储存的数字值减一, key对应的value必须是数字字符串 "1234"
$ INCR key 	# 将 key 中储存的数字值增一, key对应的value必须是数字字符串 "1234"

# 计算值长度
$ STRLEN key # 计算key对应的value的长度

    • List类型 - 存储多个字符串

      key -> string
value -> ["hello", "world", "11", "22"]

# 组织键值对
$ LPUSH key value [value ...]#相当于链表的头插,如果key不存在,一个空列表会被创建并执行LPUSH操作
$ RPUSH key value [value ...]#相当于链表的尾插,类似RPUSH
$ LPUSHX key value # 相当于链表的头插,但和LPUSH 命令相反,当 key 不存在时,LPUSHX 命令什么也不做
$ RPUSHX key value # 类似RPUSHX

# 查找
$ LINDEX key index # 返回列表 key 中，下标为 index 的元素

# 遍历
$ LRANGE key start stop
	- start: 起始位置: 0
	- stop: 最后一个 -1
	
# 元素的删除
$ LPOP key # 相当于链表的头删
$ RPOP key # 相当于链表的尾删
$ LREM key count value # 删除count个值为value的元素 
    • count > 0 : 从表头开始向表尾搜索，移除与 value 相等的元素，数量为 count 。
    • count < 0 : 从表尾开始向表头搜索，移除与 value 相等的元素，数量为 count 的绝对值。
    • count = 0 : 移除表中所有与 value 相等的值。

# 元素的插入
$ LINSERT key BEFORE|AFTER pivot value # 将value插入key中，位于值 pivot 之前或之后。
									   # 当 pivot 不存在于列表 key 时，不执行任何操作

#返回列表 key 的长度
$ LLEN key

    • Set类型

      # 集合中添加元素
$ SADD key member [member ...]
# 遍历
$ SMEMBERS key
# 差集
$ SDIFF key1 [key2 key3...] -> key1 - key2 - key3
# 交集
$ SINTER key [key ...]
# 并集
$ SUNION key [key ...]

    • SortedSet 类型

      key -> string
value -> ([score member], [score member], [score member])

# 添加元素
$ ZADD key score member [[score member] [score member] ...]
# 遍历 -> 升序
$ ZRANGE key start stop
# 遍历 -> 降序
$ ZREVRANGE key start stop [WITHSCORES]

# 返回有序集 key 中,score值在[min,max]的成员的数量。
$ ZCOUNT key min max

# 移除有序集 key 中的一个或多个成员，不存在的成员将被忽略
$ ZREM key member [member ...]

    • Hash类型

      key -> string
value -> ([field, value], [field, value])

# 设置数据
$ HSET key field value
$ HMSET key field value [field value ...] # 同时将多个 field-value 对设置到哈希表 key 中

# 根据key读value
$ HGET key field
$ HMGET key field [field ...] # 同时读多个key对应的value
$ HGETALL key 	# 一次性读key中存储的所有的key-value

    • Key 相关的命令

      # 查看所有的key
$ keys *

# 返回 key 所储存的值的类型
$ type key

# 删除给定的一个或多个 key 。
$ DEL key [key ...]

# 为给定 key 设置生存时间，当 key 过期时 (生存时间为 0 )，它会被自动删除
$ EXPIRE key seconds

# 移除给定 key 的生存时间，将这个 key 从『易失的』 (带生存时间 key ) 转换成『持久的』 (一个不带生存时间、永不过期的 key )。
$ PERSIST key

    # 2.3 redis配置文件

    配置文件是给redis服务器使用的

    1. 配置文件位置

      • 源码安装 -> 源码目录中redis.conf
      • sudo apt install redis
        • find /etc -name redis.conf

    2. 配置文件配置项

      $ redis-server redis.conf # 启动redis服务器的时候加载配置项

# 只有被绑定的IP地址才可以访问redis服务器
# 如果任意客户端都允许访问服务器, 注释掉该选项
bind 127.0.0.1 192.168.1.100

# 如果保护模式是开启的,远程客户端无法访问redis服务器
protected-mode yes # 建议设置为no

# reids服务器启动的时候绑定的默认端口
port 6379 

# 客户端和服务器连接超时断开  -> 0 to disable
timeout 0

# redis服务器启动之后是不是守护进程: yes-> 是, no-> 不是
daemonize no

# reids服务器以守护进程方式启动, 才生效, 文件中写入的是当前reids服务器进程的进程ID
# ./ == redis-server 进程启动时候所在的目录
pidfile ./redis.pid

# 日志级别
# debug   调试 (大量信息，对开发/测试有用)
# verbose 冗长 (许多信息很少有用，但不像调试级别那样混乱）            
# notice  注意 (适度冗长,可能是生产中需要的)
# warning 警告 (仅记录非常重要/关键的消息)
loglevel notice

# 只有reids服务器是守护进程的时候生效, 日志文件生成到什么地方
# ./ == redis-server 进程启动时候所在的目录
logfile "./redis.log"

# reids服务器中默认数据库的个数
databases 16

    # 2.4 redis数据持久化

    持久化: 数据从内存到磁盘的过程

    持久化的两种方式:

    • rdb

      • 默认的持久化方式, 默认开启

      • 磁盘文件 命名: xxx.rdb

      • 将内存数据以二进制方式写入到磁盘文件中

      • 文件比较小, 数据恢复效率高

      • 通过频率进行同步

        # 分别表示900秒(15分钟)内有1个更改，300秒(5分钟)内有10个更改以及60秒内有10000个更改即同步
save 900 1
save 300 10
save 60 10000
save "" # 关闭rdb持久化模式

# rdb磁盘文件名字
dbfilename dump.rdb

# 磁盘的持久化文件生成到那个目录
# ./ == redis-server 进程启动时候所在的目录
dir ./

    • aof

      • aof默认不开启

      • 磁盘文件: xxx.aof

      • 文件中保存的是操作命令

      • 文件比较大, 数据恢复效率低

      • 同步方式 -> 数据更安全

        # 数据同步的频率
appendfsync always  	# 只要进行了写操作,就同步数据
appendfsync everysec	# 每隔一秒同步一次
appendfsync no	   # 对大多数Linux操作系统,是每30秒进行一次fsync,将缓冲区中的数据写到磁盘上

# 控制aof持久化模式是不是开启: yes->开启, no->关闭
appendonly no

# 指定aof持久化文件的名字
appendfilename "appendonly.aof"

    1. aof和rdb能不能同时打开?

      • 可以

    2. aof和rdb能不能同时关闭?

      • 可以

    3. 两种模式同时开启, 如果要进行数据恢复, 如何选择?

      • 从效率角度考虑: rdb模式
      • 从数据安全性考虑: aof模式

    # 3 hiredis的使用

    1. hiredis的安装

      • 下载地址: http://redis.cn/clients.html#c
      • 安装
        • make
        • sudo make install

    2. hiredis API接口的使用

      • 连接数据库

        // 连接数据库
redisContext *redisConnect(const char *ip, int port);
redisContext *redisConnectWithTimeout(const char *ip, 
                                      int port,const struct timeval tv);

      • 执行redis命令函数

        // 执行redis命令
void *redisCommand(redisContext *c, const char *format, ...);

// redisCommand 函数实际的返回值类型
typedef struct redisReply {
    /* 命令执行结果的返回类型 */
    int type;  //见下表
    /* 存储执行结果返回为整数 */
    long long integer;
    /* str变量的字符串值长度 */
    size_t len;
    /* 存储命令执行结果返回是字符串, 或者错误信息 */
    char *str;
    /* 返回结果是数组, 代表数据的大小 */
    size_t elements;
    /* 存储执行结果返回是数组*/
    struct redisReply **element;
} redisReply;
redisReply a[100];
element[i]->str
        状态表示 含义
        REDIS_REPLY_STRING==1 返回值是字符串,字符串储存在redis->str当中
        REDIS_REPLY_ARRAY== 2 返回值是数组，数组大小存在redis->elements里面，数组值存储在redis->element[i]里面。数组里面存储的是指向redisReply的指针，数组里面的返回值可以通过redis->element[i]->str来访问，数组的结果里全是type==REDIS_REPLY_STRING的redisReply对象指针。
        REDIS_REPLY_INTEGER == 3 返回整数long long，从integer字段获取值
        REDIS_REPLY_NIL==4 返回值为空表示执行结果为空
        REDIS_REPLY_STATUS ==5 返回命令执行的状态，比如set foo bar 返回的状态为OK，存储在str当中 reply->str == "OK" 。
        REDIS_REPLY_ERROR ==6 命令执行错误,错误信息存放在 reply->str当中。

      • 释放资源

        // 释放资源
void freeReplyObject(void *reply);
void redisFree(redisContext *c);

    测试代码

    #include <stdio.h>
#include <hiredis.h>

int main()
{
    // 1. 连接redis服务器
    redisContext* c = redisConnect("127.0.0.1",6379);
    if(c->err != 0)
    {
        perror("redisConnet");
        return -1;
    }

    // 2. 连接成功,往服务器写数据
    // sting , hash
    void* ret = redisCommand(c,"hmset user username zhang3 passwd 123456 age 23 sex man");
    redisReply* reply = (redisReply*)ret;
    if(reply->type == REDIS_REPLY_STATUS)
    {
        printf("status:%s\n",reply->str);
    }

    // 3. 释放资源
    freeReplyObject(ret);

    // 4. 读数据
   reply = (redisReply*) redisCommand(c,"hgetall user");
   if(reply->type == REDIS_REPLY_ARRAY)
   {
        // 5.遍历数组
        for(int i = 0;i < reply->elements;i+=2)
        {
            // 6. 打印元素值
            printf("key:%s,value:%s\n",reply->element[i]->str,reply->element[i+1]->str);
        }
   }

    // 7. 释放资源
    freeReplyObject(ret);
    redisFree(c);

    return 0;
}
