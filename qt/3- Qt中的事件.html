      # 1.2 自定义控件

      • 自定义的控件对应是某一个类, 这个类需要继承某一个标准控件类

        • 继承了父类的一些属性
        • 在当前类中添加新属性

      • 使用

        • 先看自定义控件的父类类型

        • 找到对应的ui文件, 打开, 从工具栏中将一个父类类型的标准控件, 放到ui文件上

      # 2. 窗口布局

      # 2.1 布局方式

      • 水平布局
        • 所有的控件都水平排列
        • 一行多列
      • 垂直布局
        • 所有的控件都垂直排列
        • 多行一列
      • 网格布局
        • 控件以多行多列的方式进行排列

      # 2.2 如何布局

      • 从工具栏中找一个控件 - > QWidget, 放到ui界面上

      • 将要布局的控件放到QWidget窗口的内部, 通过对象树列表可以看到父子关系

      • 在QWidget上鼠标右键, 选择布局

      • 没有布局

      # 3. 事件

      • 什么是事件?

        • 用户对创建进行操作, 操作过程中会触发对应的消息
        • 触发的消息会被操作系统捕捉到, 放到应用程序对应的消息队列中
        • 每个启动的程序都有一个应用程序对象, 应用程序对象进行消息循环
          • 从消息队列中将消息取出
          • 处理
            • Qt中封装了一系列的消息处理函数, 都有默认的处理动作
            • 只要某个消息被触发, 对应的处理函数就会被调用
              • 调用的函数在Qt中国称之为事件处理函数
            • 如果默认的事件处理函数不能满足需求, 程序猿就必须重写这些函数

      • Qt中的事件处理函数

        • 窗口类中受保护的虚函数
          • 只能在类内部使用, 调用是Qt框架做的
          • 父类中为虚函数, 子类中重写这些虚函数, 可以实现多态

      • 举例

        // 鼠标按下
void MyLabel::mousePressEvent(QMouseEvent *ev)
{
    // 如果按下的鼠标左键
    if(ev->button() == Qt::LeftButton)
    {
        QString str = QString("x坐标: %1, y坐标: %2").arg(ev->pos().x()).arg(ev->pos().y());
        this->setText(str);
    }
}
// 鼠标释放
void MyLabel::mouseReleaseEvent(QMouseEvent *ev)
{
    // button()函数判断瞬间的状态
    if(ev->button() == Qt::LeftButton)
    {
        this->setText("鼠标键被释放 -> 左键");
    }
    else if(ev->button() == Qt::RightButton)
    {
        this->setText("鼠标键被释放 -> 右键");
    }
    else if(ev->button() == Qt::MidButton)
    {
        this->setText("鼠标键被释放 -> 中键");
    }
}
// 鼠标移动
void MyLabel::mouseMoveEvent(QMouseEvent *ev)
{
    // 鼠标移动是持续的状态
    // 判断得到复合键值中有没有鼠标左键
    // 101 == 1
    // 110
    // 001
    // 000
    if(ev->buttons() & Qt::LeftButton)
    {
        QString str = QString("x坐标: %1, y坐标: %2").arg(ev->pos().x()).arg(ev->pos().y());
        this->setText(str);
    }
}

      # 4. 绘图

      • QPainter

        // 构造函数
QPainter::QPainter(QPaintDevice *device);
// QPaintDevice: 绘图设备, 图像的载体

      • 哪些类可以作为绘图设备使用

        • QWidget
        • QPixmap
        • QImage

      • 如何画图

        在API里边有一些列的 drawxxxx();

      • QImage -> 加载磁盘图片到内存中

        // fileName: 磁盘图片的路径
QImage::QImage(const QString &fileName, const char *format = Q_NULLPTR);

        • 将图片加载到内存, 可以进行数据传输

          • 数据格式与平台无关

          • 可以修改图片的像素

          • 多线程的场景中使用

      • QPixmap

        • 主要用于图片的显示 -> 显示到窗口中
          • 效果好, 效率高
        • 为了让显示效果更好, 在不同平台做了不同的优化
        • 不能修改图片的像素
        • 不能在多线程中使用
