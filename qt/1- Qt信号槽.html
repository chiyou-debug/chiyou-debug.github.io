    # Qt信号槽

    # 1. Qt介绍

    # 1.1 Qt概述

    • Qt

      • Q - T
      • /ˈkjuːt/

    • 是一个跨平台的C++应用程序开发框架

      • java -> java虚拟机
      • 在不同的平台上封装了不同的平台的API

    • 广泛用于开发GUI程序，这种情况下又被称为部件工具箱。

      • GUI: Graphical User Interface

        • G - U - I
        • /gu:i/

    • 也可用于开发非GUI程序，比如控制台工具和服务器。

    # 1.2 优点和案例

    • Qt的优点

      • 跨平台，几乎支持所有的平台

      • 接口简单，容易上手，学习QT框架对学习其他框架有参考意义。

      • 一定程度上简化了内存回收机制

      • 开发效率高，能够快速的构建应用程序。

      • 有很好的社区氛围，市场份额在缓慢上升。

      • 可以进行嵌入式开发。

    • Qt相关案例

      • VirtualBox：虚拟机软件。
      • VLC多媒体播放器：一个体积小巧、功能强大的开源媒体播放器。
      • YY语音：又名“歪歪语音”，是一个可以进行在线多人语音聊天和语音会议的免费软件。在中国大陆拥有庞大的用户群。
      • 咪咕音乐：咪咕音乐是中国移动倾力打造的正版音乐播放器
      • WPS Office：金山公司（Kingsoft）出品的办公软件，与微软Office兼容性良好，个人版免费。
      • Skype：一个使用人数众多的基于P2P的VOIP聊天软件。

#endif // MYTEST_H

// .cpp
// QT中类的构造函数的作用: 对关联的UI界面进行数据初始化

MyTest::~MyTest()
{
    delete ui;
}

    • 坐标原点

        • x轴向右递增
        • y轴向下递增

    # 3.2 窗口显示

    • QT中的窗口是嵌套显示
      • 外层-> 父窗口
      • 内层-> 子窗口
    • 窗口显示的时候使用是相对坐标
      • 相对于父窗口的坐标原点
    • 窗口如何显示
      • 如果有父窗口
        • 父窗口如果被显示了, 子窗口会自动显示
      • 没有父窗口
        • 窗口如果要显示, 必须手动调用窗口的show()方法

      • 满足第一个条件之后, 创建当前对象的时候, 必须要指定父对象

    # 4. 信号槽

    # 4.1 概述

    • 什么是信号 ?

      • 触发的事件, 会发自动发出一个通知
      • 由某一个对象发出的, 这个对象感知到对应的事件被触发, 就发出一个信号

    • 什么是槽 ?

      • 就是一个处理动作, 处理某个信号
      • 处理动作也是属于某个对象

    • 信号槽的本质?

      • 信号, 在Qt中是一个函数, 只有声明, 没有定义(不需要实现)
      • 槽, 在Qt中也是以函数, 需要有声明和实现

    • 信号槽的使用?

      • 对一个对象(对象1)进行操作, 这个对象会发出一个信号

      • 如果其他对象(对象2)对对象1发出的信号感兴趣, 需要在所对应的类中编写以处理函数, 这个函数就是槽函数

        class A
{
public:
    void print();
}
A *a = new A;
a->print();

      • 信号函数和槽函数之间默认没有关联, 需要使用 connect函数将二者进行关联

    • 语法

      connect(sender, &Sender::signal, receiver, &Receiver::slot);
参数: 
	- sender: 这是一对象的地址, 发出信号的对象
	- signal: sender对象发出的信号
		- &Sender::signal
		 - Sender: 发出信号的对象对应的类的名字
		 - signal: 发出的具体的信号
	- receiver: 这是一对象的地址, 接收发出的信号
	- slot: 信号接受者对信号的处理动作

    # 4.2 使用

    • 标准信号槽的使用

      • 类自带的信号和槽函数

      • 在窗口上放一个按钮, 当点击按钮的时候关闭当前窗口

        // 信号发出者 -> 按钮
// 外层窗口处理信号, 处理动作 -> 关闭当前窗口
// 管理者两个对象: 按钮对象, 外层的窗口对象
// 注册信号槽
connect(button, &QPushButton::clicked, bigWin, &MyTest::close);

    • 自定义信号槽的使用

      • 自定义信号

        • 返回值: void

        • 信号是函数, 因此可以携带参数: 可以重载

        • 信号函数需要使用signals关键字声明

        • 信号函数不需要定义, 只有声明

        • 发送自定义信号:

          emit 自定义的函数();
emit mysignal();

      • 自定义槽函数

        • 返回值: void, 和信号对应

        • 参数: 可以重载, 自定义槽函数目的是为了处理某一个信号

          • 槽函数和连接的信号函数参数必须一一对应

            • 个数, 类型, 顺序都一样

          • 槽函数的参数个数 <= 信号的参数个数

        • // 信号的参数是用于传递数据的, 数据从信号中 -> 槽函数中
// -------- 不带参数
// 要发出的信号
signals:
	void myTest();
// 要处理信号的槽函数
void mySlot();

// ----------- 带参数
// 要发出的信号
signals:
	// 谁发送信号, 谁给形参初始化
	void myTest(int a, QString text);
// 要处理信号的槽函数 -> 接收信号传递的数据, 参数依次匹配, 要求数据类型一致
void mySlot(int a, QString text);	// ok

// ----------- 带参数
// 要发出的信号
signals:
	// 谁发送信号, 谁给形参初始化
	void myTest(int a, QString text);
// 要处理信号的槽函数 -> 接收信号传递的数据, 参数依次匹配, 要求数据类型一致
void mySlot(int a);	// ok

// ----------- 带参数
// 要发出的信号
signals:
	// 谁发送信号, 谁给形参初始化
	void myTest(int a, QString text);
// 要处理信号的槽函数 -> 接收信号传递的数据, 参数依次匹配, 要求数据类型一致
void mySlot(int a, QString text, int c);	// error

// ----------- 带参数
// 要发出的信号
signals:
	// 谁发送信号, 谁给形参初始化
	void myTest(int a, QString text);
// 要处理信号的槽函数 -> 接收信号传递的数据, 参数依次匹配, 要求数据类型一致
void mySlot(QString text);	// error

    # 5. Lambda表达式

    • 匿名函数
    • 没有函数名, 只有函数体

    # 5.1 语法

    [capture](params) opt -> ret {body;};
	- []: 匿名函数的开始
	- (): 参数列表 == 和c++是一样的
	- opt: 可选项, 不使用可以忽略不写
		- mutable
        - throws
    - ->返回值: 函数的返回值, 可以省略, c++11返回值类型可以自动推导
    - {}: 函数体
    
// 例子
[](){
	int a = 10;
	printf("%d\n", a);
	return a;
}

    • 变量的捕获方式

      • []- 不捕捉

      • [&]- 引用的方式使用外部变量

      • [=]- 拷贝的方式使用....., 函数体内使用的是副本

      • [=, &foo] - foo使用引用, 其余是拷贝的方式

      • [bar] - 在函数体内部使用拷贝的方式访问bar变量

      • [this] - 在函数内部使用类的成员函数和成员变量

    # 5.2 注意事项

    • lambda表达式无法修改通过复制方式捕获的外部变量

    • 如果需要修改, 请使用引用的方式捕获外部变量

    • 如果希望修改按值捕获的外部变量需要使用mutable

      [=] mutable {return a++;}

    • int a = 1, b=9;

      // a是一个拷贝的副本, 默认是const类型
[=] {return a++;}; // error
