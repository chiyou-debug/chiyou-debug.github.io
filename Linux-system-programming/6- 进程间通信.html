      # 进程间通信

      # 1. 概述

      • IPC - InterProcess Communication

        进程是一个独立的资源分配单元，不同进程（这里所说的进程通常指的是用户进程）之间的资源是独立的，没有关联，不能在一个进程中直接访问另一个进程的资源。

        但是，进程不是孤立的，不同的进程需要进行信息的交互和状态的传递等，因此需要进程间通信( IPC：Inter Processes Communication )。

        进程间通信的目的：

        • 数据传输：一个进程需要将它的数据发送给另一个进程。

        • 通知事件：一个进程需要向另一个或一组进程发送消息，通知它（它们）发生了某种事件（如进程终止时要通知父进程）。

        • 资源共享：多个进程之间共享同样的资源。为了做到这一点，需要内核提供互斥和同步机制。

        • 进程控制：有些进程希望完全控制另一个进程的执行（如 Debug 进程），此时控制进程希望能够拦截另一个进程的所有陷入和异常，并能够及时知道它的状态改变。

      • 进程间通信的方式?

        • 管道 (使用最简单)
          • 有名管道 -> 无学员关系
          • 匿名管道 -> 有血缘关系
        • 信号 (开销最小)
          • 强烈不推荐 (其一：信号优先级很高，会打断程序的执行顺序；其二：信号携带的信息量极少)
        • 内存映射区 (无血缘关系)
        • 本地套接字 (最稳定)
        • 其他：共享内存、消息队列、文件

      # 2. 管道

      它是UNIX 系统 IPC(进程间通信)的最古老形式，所有的 UNIX 系统都支持这种通信机制。

      • 实现原理：
        • 内核借助环形队列，使用内核缓冲区实现，默认大小为4K
      • 特质：
        • 伪文件
        • 管道中的数据只能一次读取
        • 数据在管道中，只能单向流动
      • 局限性：
        • 自己写，不能自己读
        • 数据不可反复读
        • 半双工通信

      # 2.1 匿名管道

      • 匿名管道的特点?

        • 这个管道没有名字
        • 因为底层的数据结构是队列，因此数据是先进先出且只能读一次, 读完之后数据在管道中就消失了
        • 管道的属性默认是阻塞的 -> 默认阻塞是在读写两端都打开的时候
          • 读阻塞
            • 有数据, 读数据
            • 当管道中没有数据的时候, 读端阻塞
          • 写阻塞
            • 当管道被写满, 写端阻塞
            • 只要管道没满, 就可以写
        • 只能进行有血缘关系的进程间通信

      • 创建匿名管道

        #include <unistd.h>
// 创建并打开匿名管道
// pipefd是传出参数, 对应的是管道的两端
// pipefd[0] -> 读端, pipefd[1] -> 写端
int pipe(int pipefd[2]);
返回值: 
	成功: 0
    失败: -1

      • 使用匿名管道实现进程间通信

        /* 
    父子进程间通信, 实现 ps aux 
    	- 子进程执行shell命令: ps ·	
    	- 父进程中输出命令结果
    注意事项: 保证数据是单向流动的
    	- 父进程读管道: 关闭写端
    	- 子进程写管道: 关闭读端
*/ 

处理流程:
	- 创建匿名管道: fd0, fd1
	- 创建子进程 -> fork();
		- fd0, fd1被拷贝到子进程中
		- 父子进程可以操作同一个管道, 通过fd0, fd1
	- 在子进程中调用 execlp("ps", "ps", "aux", NULL);
		- 得到的结果会被打印到终端, 不好操作
		- 数据不能打印到终端 -> 重定向: 将终端的标准输出重定向到管道的写端
			- dup2(fd[1], STDOUT_FILENO);
	- 父进程通过 fd0 读管道就可以将数据取出

        示例

        #include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <string.h>

int main()
{
    //1、创建匿名管道
    int fd[2];
    int ret = pipe(fd);
    if(ret == -1)
    {
        perror("pipe");
        return -1;
    }

    //2、创建子进程
    pid_t pid = fork();
    if(pid > 0)
    {
        // 父进程,读管道,关闭写端
        close(fd[1]);
        char buf[1024];
        int len = 0;
        while((len=read(fd[0],buf,sizeof(buf)))!=0)
        {
            // printf("%s\n",buf); //err,段错误
            write(STDOUT_FILENO,buf,len);
        }
        printf("\n");
        close(fd[0]);
    }
    else if(pid == 0)
    {
        //关闭读端
        close(fd[0]);
        //重定向: 将标准输出 -> 管道的写端
        dup2(fd[1],STDOUT_FILENO);
        //子进程 ps aux
        execlp("ps","ps","aux",NULL);
        perror("ps");
    }

    return 0;
}

        管道的读写行为?

        ① 读管道： 1. 管道中有数据，read返回实际读到的字节数。

        ​ 2. 管道中无数据：

        ​ (1) 管道写端被全部关闭，read返回0 (好像读到文件结尾)

        ​ (2) 写端没有全部被关闭，read阻塞等待(不久的将来可能有数据递达，此时会让出cpu)

        ② 写管道： 1. 管道读端全部被关闭，管道破裂进程异常终止(也可使用捕捉SIGPIPE信号，使进程不终止)

        ​ 2. 管道读端没有全部关闭：

        ​ (1) 管道已满，write阻塞。

        ​ (2) 管道未满，write将数据写入，并返回实际写入的字节数。

      • 如何设置管道非阻塞?

        #include <unistd.h>
#include <fcntl.h>

int fcntl(int fd, int cmd, ... /* arg */ );

// cmd 取值: F_GETFL / F_SETFL
// 非阻塞的属性: O_NONBLOCK

// 获取当前fd对应的flags属性
// 设置读端非阻塞
int flag = fcntl(fd[0], F_GETFL);
// 将 O_NONBLOCK 设置 到flag属性中
flag = flag | O_NONBLOCK;	// flag |= O_NONBLOCK;
// 将flag属性设置给 fd[0]
fcntl(fd[0], F_SETFL, flag);

      # 2.2 有名管道

      匿名管道的优劣

      ​ 优点：简单，相比信号，套接字实现进程间通信，简单很多。

      ​ 缺点：1. 只能单向通信，双向通信需建立两个管道。

      ​ 2. 只能用于父子、兄弟进程(有共同祖先)间通信。该问题后来使用fifo有名管道解决。

      • FIFO

        • FIFO常被称为命名管道，以区分管道(pipe)。管道(pipe)只能用于“有血缘关系”的进程间。但通过FIFO，不相关的进程也能交换数据。
        • FIFO是Linux基础文件类型中的一种。但，FIFO文件在磁盘上没有数据块，仅仅用来标识内核中一条通道。各进程可以打开这个文件进行read/write，实际上是在读写内核通道，这样就实现了进程间通信。

      • 创建方式 -> 两种

        ## 通过命令创建
$ mkfifo 管道名

        // 通过函数
#include <sys/types.h>
#include <sys/stat.h>

int mkfifo(const char *pathname, mode_t mode);
	参数:
		- pathname: 要创建的管道的名字
		- mode: 创建的管道文件的权限, 指定一个八进制的数
			文件的最终权限: (mode & ~umask)

      • 使用fifo进行没有血缘关系的进程间通信

        创建一个有名管道: fifotest

        mkfifo fifotest

        • 进程A -> 读操作

          #include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/stat.h>
#include <string.h>
#include <fcntl.h>

// 读管道的进程
int main()
{
    // 1. 打开磁盘上的管道文件
    int wrfd = open("fifotest", O_RDONLY);
    if(wrfd == -1){
        perror("open");
        return -1;
    }

    // 2. 从管道中read数据
    char buf[1024] ;
    while(1)
    {
        memset(buf, 0, sizeof(buf));
        int len = read(wrfd, buf, sizeof(buf));
        printf("recv buf: %s, data len: %d\n", buf, len);
        if(len == 0){
            printf("管道数据已经读完了, 写端已经关闭\n");
            break;
        }
    }
    close(wrfd);

    return 0;
}

        • 进程B -> 写操作

          #include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <string.h>
#include <fcntl.h>

// 写管道的进程
int main()
{
    // 1. 打开磁盘上的管道文件
    int wrfd = open("fifotest", O_WRONLY);
    if(wrfd == -1){
        perror("open");
        return -1;
    }

    // 2. 往管道中写数据
    char buf[1024];
    int number = 0;
    while(number <= 100)
    {
        sprintf(buf, "我是写管道的进程 -- %d", number++);
        printf("%s\n", buf);
        write(wrfd, buf, strlen(buf));
        sleep(1);
    }
    close(wrfd);

    return 0;
}

      • 示例:父子进程实现 ls | wc-l

        #include<stdio.h>
#include<stdlib.h>
#include<string.h>
#include<unistd.h>
#include<pthread.h>

int main()
{
    int fd[2];
    int ret;
    pid_t pid;

    ret = pipe(fd);

    if(ret ==-1 ){
        perror("pipe error");
        exit(1);
    }

    pid = fork();
    if(pid == -1)
    {
        perror("fork error");
    }else if(pid > 0){
        close(fd[1]);
        dup2(fd[0],STDIN_FILENO);
        execlp("wc","wc","-l",NULL);
    }else if(pid == 0){
        close(fd[0]);
        dup2(fd[1],STDOUT_FILENO);
        execlp("ls","ls",NULL);
    }

    return 0;

      # 3. 内存映射

      # 3.1 文件完成进程间通信

      使用文件也可以完成IPC，理论依据是，fork后，父子进程共享文件描述符。也就共享打开的文件

      • 编程测试，父子进程共享打开的文件。借助文件进行进程间通信

        fork_shared_fd.c

        /* 
 *父子进程共享打开的文件描述符------使用文件完成进程间通信.
 */
#include <stdio.h>
#include <unistd.h>
#include <string.h>
#include <stdlib.h>
#include <fcntl.h>
#include <sys/wait.h>

void sys_err(const char *str){
	perror(str);
	exit(1);
}

int main(void)
{
	int fd1, fd2; pid_t pid;
	char buf[1024];
	char *str = "---------test for shared fd in parent child process-----\n";

	pid = fork();

	if (pid < 0) 
		sys_err("fork err");
	else if (pid == 0) {
		fd1 = open("test.txt", O_RDWR|O_CREAT|O_TRUNC, 0644);
		if (fd1 < 0) 
			sys_err("open err");

		sleep(3);					// 如果睡眠,父进程,会读到数据吗?

		write(fd1, str, strlen(str));
		printf("child wrote over...\n");
	} else {
		fd2 = open("test.txt", O_RDONLY);
		if (fd2 < 0) 
			sys_err("fork err");
		
        sleep(1);                   //保证子进程写入数据
        
		int len = read(fd2, buf, sizeof(buf));
		len = write(STDOUT_FILENO, buf, len);
		printf("---- parent write len = %d\n", len);

		wait(NULL);
	}

	return 0;
}

      # 3.2 mmap内存映射

      存储映射I/O (Memory-mapped I/O) 使一个磁盘文件与存储空间中的一个缓冲区相映射。于是当从缓冲区中取数据，就相当于读文件中的相应字节。于此类似，将数据存入缓冲区，则相应的字节就自动写入文件。这样，就可在不适用read和write函数的情况下，使用地址（指针）完成I/O操作。

      使用这种方法，首先应通知内核，将一个指定文件映射到存储区域中,这个映射工作可以通过mmap函数实现.

      有一个磁盘文件, 某个进程通过调用一个函数 mmap可以在当前进程的虚拟地址空间中申请一块内存, 将磁盘文件中的内容映射到这块内存中. 这块内存就叫内存映射区

      两个进程通过内存映射区实现进程间通信, 每个进程中都有一块属于自己的内存映射区

      • 可以实现没有血缘关系的进程间通信

      • 使用的步骤:

        1. 在当前进程的虚拟地址空间中创建一块映射内存
	- mmap
	- 创建出的内存在虚拟地址空间 (位于动态库加载区)
    - 每个进程都可以创建一个属于自己的虚拟地址空间
    - 多个内存映射区要实现数据同步, 必须要操作同一个磁盘文件, 并且形成映射关系
    - 某一个进程对映射区数据进行了修改, 数据被同步到磁盘, 磁盘数据又同步到了其他进程的映射区中
2. 对映射区数据修改 == 内存操作

      • 函数原型

        #include <sys/mman.h>
// 创建内存映射区
void *mmap(void *addr, size_t length, int prot, int flags, int fd, off_t offset);
	参数:
		- addr: 在动态库加载区的什么位置创建一块映射内存, NULL -> 委托内核指定
		- length: 创建的内存映射区大小, 指定我们想要的大小即可
			- 实际分配的大小是4k整数倍(>0)  -> 内存页的整数倍
            - 这个参数长度不能为0
        - prot: 进程对内存映射区的操作权限, 读权限是必须要指定的
        	- PROT_READ: 读权限
        	- PROT_WRITE: 写权限
        	- PROT_READ | PROT_WRITE: 读写权限
        - flags: 实现进程间通信,必须指定 MAP_SHARED
			- MAP_SHARED: 数据共享, 当映射区数据被修改, 数据才会同步到磁盘文件
			- MAP_PRIVATE: 数据私有, 当映射区数据被修改, 数据不会同步到磁盘文件
		- fd: 磁盘文件对应的文件描述符
			- 内存映射区和这个fd对应的磁盘文件进行映射(这个文件的大小不能为0)
			- 得到fd需要open(), 在open的时候需要指定对文件的操作权限
				- 这个权限和prot参数的权限对应即可
		- offset: 在映射的时候对磁盘文件进行偏移
			- 这个参数必须是4k的整数倍
			- 不偏移指定为0
	返回值:
		- 成功: 内存映射区的起始地址
		- 失败: MAP_FAILED (that is, (void *) -1) )

        // 释放内存映射区
int munmap(void *addr, size_t length);
	参数:
		- addr: 内存映射区的首地址
		- length: 和 mmap 第二个参数值相同即可
	返回值:
		成功: 0
        失败: -1

      • 进程间通信 -> 内存操作因此不阻塞

        • 实现进程间通信的核心:
          • 多个进程进行内存映射, 使用的磁盘文件必须是同一个, 通过这个磁盘文件同步各个映射区的数据

      • 有血缘关系

          /*
  	1. 在父进程中创建内存映射区
  	2. 通过父进程创建子进程 -> fork()
  		- 父进程的虚拟地址空间被拷贝 -> 子进程
  		- 内存映射区也被拷贝到子进程中
  	3. 父子进程都拿到了自己内存映射区起始地址
  	4. 读写内存
  		父进程 -> 写内存
  		子进程 -> 读内存
  */

        示例：parent-child-mmap.c

        #include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <string.h>
#include <sys/mman.h>
#include <fcntl.h>

int main()
{
    // 1.创建内存映射区
    int fd = open("test.txt",O_RDWR);
    if(fd == -1)
    {
        perror("open");
        exit(0);
    }

    void* ptr = mmap(NULL,100,PROT_READ|PROT_WRITE,MAP_SHARED,fd,0);
    if(ptr == MAP_FAILED)
    {
        perror("mmap");
        exit(0);
    }

    // 2.创建子进程
    pid_t pid = fork();
    if(pid == 0)
    {
        // 子进程
        // 因为mmap非阻塞,因此要等到父进程写完
        sleep(1);
        printf("父进程写入的数据:%s\n",(char*)ptr);
    }
    else if(pid > 0)
    {
        // 父进程
        // 写内存
        char* p = "你是我的儿子吗,我是你爹...";
        memcpy(ptr,p,strlen(p)+1);
    }

    //释放内存映射区
    munmap(ptr,100);

    close(fd);

    return 0;
}

      • 没有血缘关系

        /*
 进程A: -> 写
  	1. 创建一个内存映射区, 和磁盘的 文件 test.txt 进行映射
  	2. 得到一个映射区起始地址
  	3. 通过起始内存地址, 写数据
  	4. 释放内存映射区
  	
 进程B: -> 读
  	1. 创建一个内存映射区, 和磁盘的 文件 test.txt 进行映射
  	2. 得到一个映射区起始地址
  	3. 通过起始内存地址, 读数据
  	4. 释放内存映射区
  */

      • 思考问题

        1. 如果对mmap的返回值(ptr)做++操作(ptr++), munmap是否能够成功?
    - 不能
    - 需要用一个指针将映射区起始地址保存下来，方便后边进行释放
2. 如果open时O_RDONLY, mmap时prot参数指定PROT_READ | PROT_WRITE会怎样?
	- open的时候也应该指定写权限
    - 一般来说,open的时候指定的权限应该和prot对应的权限相同
3. 如果文件偏移量为1000会怎样?
	- 函数调用失败，无效的参数，参数值必须是4k的整数倍或者0(表示不偏移)
4. 可以open的时候O_CREAT一个新文件来创建映射区吗?
	- 因为创建出的新文件大小为0，和映射区进行映射的文件大小必须>0
	- 可以在文件被创建之后使用`int ftruncate(int fd, off_t length);`对文件进行扩展
5. mmap后关闭文件描述符，对mmap映射有没有影响？
	- 没有影响
6. 对mmap的返回值ptr越界操作会怎样？
	- 段错误
7. mmap什么情况下会调用失败?
    - 第二个参数：length < 0
    - 最后一个参数：offset，必须是4k整数倍
